### 相关API
#### 1. 添加人员（仅限管理员）
用于添加访问账户，可选用户所属角色组，关系到访问权限。新添加的用户的密码都是`123456`，并在登录时以`is_simple`标记用户是否修改了默认密码。
```javascript
{
    url : "127.0.0.1:7999/api/accounts"
    method : POST
    data : {
        "username"  # 用户名
        "role"      # 角色(粮油公司0, 教体局1, 学校2)
    }
    return : {
        "msg":     # 消息
        "data":    # 数据
        "code":    # 状态码
    }
}
```
示例：
```javascript
{
    url : "http://127.0.0.1:7999/api/accounts/"
    method : POST
    data : {
        "username" : "ldzx",
        "role" : 2
    }
    return : {
        "msg": "Created successfully",
        "data": {
            "id": 8,
            "username": "ldzx",
            "is_active": true,
            "role": 2
        },
        "code": 201
    }   
}
```
#### 2. 修改密码（用户）
用于用户修改自己的密码，仅能修改自己的密码
```javascript
{
    url : http://127.0.0.1:8000/api/accounts/<id>/
    method : PUT
    data: {
        "username" # id对应的用户名
        "password" # 修改后的密码
    }
    return : {
        "msg"
        "data":{
            "id"
            "username"
            "is_active"
            "role"
        }
        "code"
    }
}
```
示例
```javascript
{
    url : http://127.0.0.1:8000/api/accounts/4/

    data : {
        "username":"ldzx",
```

```javascript
{
    url : http://127.0.0.1:7999/api/login/
    method : POST
    data : {
        "username" # 用户名
        "password" # 密码
    }
    return : {
        "msg"
        "data":{
            "refresh"   # 用于刷新token，暂时没用
            "access"    # token，用于权限管理

            "is_admin"  # 标记是否为管理员

        "code"
    }
}
```

**权限：仅粮油公司组(`role=0`)**

<font color=red> 添加商品，会赋予商品一个初始价格，刚创建时商品的状态`status` 设置为`false`（目前商品的状态没用），当第一次提交报价后修改状态为`true`。在价格周期创建后创建的商品，商品都属于未上架状态，需等待下一次周期创建。</font>

```javascript
    return : {
        "msg",
        "data":{
            "id"
            "name"
            "image"         
            "description"
            "category"
            "price"
            "unit"
        }，
        "code"
    }
}
```

## 价格相关
### <font color=red>商品价格API</font>
现在修改为不由粮油公司组(`role=0`)创建价格，而是教体局组(`role=1`)创建价格周期。当价格周期被创建时，会自动为所有商品创建当前价格周期对应的一个价格对象`price`。`price`的初始状态`status`为`0`，表示为未申报。状态总共有4种，`0：未申报,1：未审核，2：已审核，3：已拒绝`。
#### 1. 查看所有价格
**权限：仅粮油公司组(`role=0`)**  
查看目前已有的价格对象，可使用`product_id`、`status`、`cycle_id`进行筛选。
```javascript
{
    url : http://127.0.0.1:8000/api/price/
    method : GET
    params: {
        page
        page_size
        product_id    # 商品ID，筛选出某个商品的所有价格对象
        status        # 商品状态，筛选出某个状态下的所有价格对象
        cycle_id      # 周期ID，筛选出某个周期下的所有价格对象
    }
    return : {
        "msg"
        "data": {
            "count"
            "next"
            "previous"
            "results": [
                {
                    "id"                    # 价格对象的ID,修改报价时需使用
                    "price"                 # 当前价格
                    "start_date"            # 价格开始日期
                    "end_date"              # 价格结束日期
                    "status"                # 价格状态
                    "creater_id"            # 价格创建者ID
                    "create_time"           # 价格创建时间
                    "reviewer_id"           # 审核人ID
                    "review_time"           # 审核时间
                    "product_id"            # 商品ID
                    "cycle_id"              # 价格周期ID
                    "product_name"          # 商品名称
                    "product_unit"          # 商品单位
                    "product_category"      # 商品类别
                    "product_description"   # 商品描述
                }
            ]
        },
        "code"
    }
}
```

#### 2. 修改商品价格
**权限：仅粮油公司组(`role=0`)**

```javascript
{

            "product_id"
            "cycle_id"
            "product_name"
            "product_unit"

        "code"
```

#### 2. 批准某个价格请求
**权限：仅教体局组(`role=1`)**

批准某个价格请求,url中传入的是上一点中的**价格请求的`id`**,不是商品`id`。批准后当前所请求的价格的`status`将会设置为`true`，并将商品之前的价格状态设置为`false`。
```javascript
{
    url : http://127.0.0.1:8000/api/priceReq/<id>/review/
    method : POST
    return : {
        "msg"
        "data": null,
        "code"
    }
}
```
示例
```javascript
{
    url : http://127.0.0.1:8000/api/priceReq/<id>/review/
    method : POST
    return : {
        "msg": "Price Review successfully.",
        "data": null,
        "code": 200
    }
}
```
